# Standards and specification review

Reviewed against `docs/SPEC.md` after implementation.

- Target and packaging: WPF, .NET 10, win-x64, framework-dependent single-file output configured.
- Version metadata: product version 1.3.0 and assembly/file version 1.3.0.0; the wire protocol remains v1 and now includes `shortcutChord`.
- Repository isolation: source, tests, docs, TDD log, runtime-data, and artifacts are repository-local; `.tools`, `.scratch`, runtime-data, build outputs, and binary test reports are ignored as specified. Both release projects disable debug symbols and map source roots to deterministic virtual paths so the published single-file EXE contains no local user or checkout path.
- Transport: TLS over TCP, default port 39421, TCP_NODELAY, single authenticated client with busy response, strict UTF-8 JSON Lines, 65,536-byte frame bound.
- Connection hardening: TLS through the first auth frame has a 10-second linked deadline; pending authentication is capped at four, permits are released in `finally`, authenticated command streams have no auth deadline, and stop awaits every tracked session.
- Shutdown bounds: `replaceTail.deleteCodePoints` is capped at 1,024 and dispatch observes session cancellation between Backspaces and between text-injection batches; an accepted `TcpClient` enters its outer ownership scope before `NoDelay`, permit waits, or any other cancellable operation, closing the accept/stop leak window.
- Identity/auth: persistent self-signed certificate and 16-character Base32 key; DER SHA-256 fingerprint and HMAC proof follow the v1 formula; proof comparison is constant-time.
- Strictness: required/duplicate/unknown fields, types, enum values, versions, malformed JSON, invalid UTF-8, and oversized lines are rejected.
- Input: public command boundary maps all specified commands; Unicode injection sends each UTF-16 code unit or normalized Enter as an independent down/up batch, observes session cancellation between batches, and waits 1 ms after every successful text batch so native injection preserves every code unit; supplementary code points still emit two ordered surrogate-pair batches. Game-key state maps W/A/S/D to fixed scan codes with idempotent down/up tracking and best-effort release on session cleanup. Fixed system shortcuts and validated user modifier chords use single-batch scan-code sequences with reverse release and failure compensation. Real-device production-path evidence showed that tight sends and scheduler yield could both repeat the final code unit, while a 1 ms wait preserved input order. Both the pacing seam and system-sleep seam are replaceable in tests, and pacing does not affect key, pointer, button, or wheel paths; real user32 `SendInput` is crossed in the end-to-end boundary test.
- Safety/lifecycle: the manifest uses standard UAC elevation with `requireAdministrator` and `uiAccess="false"`; there is no service, scheduled task, autostart, silent elevation, or hidden self-start, and the UAC secure desktop remains out of scope.
