# 局域网无线键盘 / LAN Wireless Keyboard

Android 客户端通过局域网连接 Windows `VirtualKeyboardReceiver`，把手机原生输入法、方向键、游戏移动摇杆、触控板和鼠标按键转换为 Windows 输入事件。

## 使用

1. 启动 Windows `VirtualKeyboardReceiver.exe`，并在系统提示时允许其以管理员权限运行。
2. 打开手机应用，在设置中填写电脑局域网 IPv4、端口和接收端显示的 16 位配对码。
3. 状态显示“已连接”后，直接在原生输入框中输入或使用控制区。左侧模块发送 Windows 方向键；右侧游戏模块以 W/A/S/D 标明四个移动键，中央八向摇杆会按住相应按键，斜向可同时按住两个键。触控板支持单指移动光标、轻点左键、第二次抬手时执行左键双击；长按一秒会进入左键按住状态并震动一次，轻点后再次按住拖动仍作为保底手势。普通触摸和移动光标不会震动。独立按钮或双指轻点触发右键，双指滑动或右侧独立滚轮区负责滚动，左上角按钮发送 Esc。
4. 顶部可以在“同步模式”和“异步模式”之间左右切换。两种模式的输入栏均保持一行高；同步模式会实时发送输入，并在输入栏右侧提供地球键和“中/EN”键。默认分别发送 `Win + Space` 和 `Shift`，也可在设置中改为常见的 Windows/macOS 快捷键。异步模式把内容保留在手机，点击右侧发送图标才会整段发送；换行仍会保留在草稿中，输入栏以单行视口滚动显示。输入栏为空时，Backspace、Enter 和 Space 会直接作用于电脑。
5. 底部快捷键栏可选择 Shift、Control、Alt/Opt 和 Win/Cmd。轻点后下一次键盘按键会与所选修饰键一起发送；长按一秒会把当前选择转为半锁定并保持到再次点击。键盘弹出时可用箭头展开或收起快捷键栏。

界面沿用单页结构，不使用阴影、描边或渐变。新安装的浅色模式默认使用 `#F2F2F2` 背景和 `#B4B4B4` 图标，同时内置深色模式；默认跟随 Android 的深色模式开关，也可以在设置中关闭跟随并强制使用浅色或深色。设置、发送、地球键、启动器和方向图标来自 MIT 许可的 Hugeicons Free Icons 4.2.2 Stroke Rounded，方向区使用四个无柄尖角图标围绕中央摇杆。

设置中可以分别选择“中/EN”和输入法切换快捷键，调节指针速度、触控板加速度、滚轮刻度、惯性和震感，关闭右侧独立滚轮区。颜色不再使用多个独立输入框，而是由一份包含 `[light]` 和 `[dark]` 的配置框架统一管理；每套方案都可调整页面背景、图标、强调色、主要文字、次要文字、输入框/面板背景和触控板背景。应用会提供字段说明与 AI 提示词，并可一键复制提示词和当前框架到 ChatGPT、Claude 等工具中协助修改。保存后立即生效并在下次启动时继续使用。

同步模式会优先把可映射的 ASCII 字符作为物理键盘扫描码发送，提高普通游戏窗口的兼容性；中文、Emoji 和无法映射的字符仍使用 Unicode。使用 Raw Input、反作弊驱动或明确拒绝 `SendInput` 的游戏可能仍不接受远程输入。

应用不提供额外的中英文模式，也不猜测手机输入法状态。搜狗中文输入时，拼音、联想和候选阶段由搜狗留在手机，选字后只发送最终中文；搜狗英文输入时，输入法交给输入框的 composing 单词会实时同步，提交空格和自动更正时再更新尾部。应用不会自动切换中英文，也不会在检测到中文时回删所谓“孤立拼音”。首次成功配对后，手机会固定接收端证书指纹；接收端证书变化会被拒绝。

应用没有云端、Service、开机广播、前台通知、WakeLock 或后台队列。Activity 离开前台时会立即关闭 TLS 连接和工作线程；断线期间的输入不会重放。

## 构建与测试

项目需要 JDK 17 和 Android SDK 36：

```bash
./gradlew clean testDebugUnitTest lintDebug assembleDebug
```

连接 Android 真机后，再运行仪器测试：

```bash
./gradlew connectedDebugAndroidTest
```

生成的可安装 APK 位于 `app/build/outputs/apk/debug/app-debug.apk`。产品与协议契约见 [docs/SPEC.md](docs/SPEC.md)，第三方素材许可见 [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md)。
